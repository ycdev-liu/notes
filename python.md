

## 一、Python 基本概念

### 1. Python 是强类型还是弱类型？动态还是静态？
- **强类型 + 动态语言**
- 类型在运行时确定
- 不支持隐式类型转换（`"1" + 1` 会报错）
---

### 2. 可变对象 vs 不可变对象
- 不可变：`int`、`float`、`str`、`tuple`
- 可变：`list`、`dict`、`set`
- 不可变对象修改会创建新对象

---

### 3. list / tuple / set / dict 区别

| 类型 | 是否有序 | 是否可变 | 是否可重复 |
|---|---|---|---|
| list | 是 | 是 | 是 |
| tuple | 是 | 否 | 是 |
| set | 否 | 是 | 否 |
| dict | 是（3.7+） | value 可变 | key 不重复 |

---

### 4. `is` 和 `==` 的区别
- `is`：判断是否为同一对象（id）
- `==`：判断值是否相等

---

## 二、函数与参数

### 5. Python 参数传递机制
- **对象引用传递**
- 可变对象在函数内修改会影响外部
- 不可变对象表现得像值传递

---

### 6. 为什么默认参数不能用可变对象？
```python
def f(x, arr=[]):
    arr.append(x)
````

* 默认参数在函数定义时创建
* 多次调用共享同一个对象

---

### 7. `*args` 和 `**kwargs`

* `*args`：接收位置参数（tuple）
* `**kwargs`：接收关键字参数（dict）

---

## 三、闭包与装饰器（高频）

### 8. 什么是闭包？

* 内层函数引用外层函数的变量
* 常用于装饰器、延迟计算

---

### 9. 装饰器本质是什么？

* 函数作为参数
* 返回一个新函数（闭包）

```python
def deco(func):
    def wrapper(*args, **kwargs):
        return func(*args, **kwargs)
    return wrapper
```

---

### 10. 多个装饰器的执行顺序

```python
@a
@b
def f():
    pass
```

等价于：

```python
f = a(b(f))
```

---

### 11. 为什么要用 `functools.wraps`？

* 保留原函数的：

  * `__name__`
  * `__doc__`

---

## 四、迭代器与生成器

### 12. 什么是迭代器？

* 实现 `__iter__` 和 `__next__`

---

### 13. 生成器的优点

* 惰性计算
* 节省内存
* 适合大数据处理

---

### 14. `yield` 和 `return` 的区别

* `yield`：返回值并保存状态
* `return`：结束函数

---

## 五、异常处理

### 15. `try / except / else / finally` 执行顺序

* 无异常：`try → else → finally`
* 有异常：`try → except → finally`

---

### 16. `raise` 的作用

* 主动抛出异常
* 自定义异常

---

## 六、面向对象（OOP）

### 17. `@staticmethod` vs `@classmethod`

| 类型           | 参数         | 场景   |
| ------------ | ---------- | ---- |
| staticmethod | 无 self/cls | 工具函数 |
| classmethod  | cls        | 工厂方法 |

---

### 18. Python 的“私有变量”

* `__name` 会被解释器改写（name mangling）
* 并非真正私有

---

### 19. 多继承的 MRO

* 使用 **C3 线性化**
* `super()` 按 MRO 顺序调用

---

## 七、内存与 GC（中高级）

### 20. Python 内存管理机制

* 引用计数
* 标记-清除
* 分代回收

---

### 21. 什么是循环引用？

* 对象相互引用，引用计数不为 0
* 由 GC 的标记-清除解决

---

## 八、并发基础（必考）

### 22. GIL 是什么？

* 全局解释器锁
* 同一时间只有一个线程执行字节码
* 限制多线程 CPU 并行

---

### 23. 线程 / 进程 / 协程使用场景

* IO 密集：多线程 / 协程
* CPU 密集：多进程

---

## 九、字符串与编码

### 24. Python 3 中的 `str` 是什么？

* Unicode
* UTF-8 用于存储和传输

---

### 25. `join` 和 `+` 拼接字符串区别

* `join` 更高效（一次性分配内存）

---

## 一、对象模型（Object Model）

### 1. Python 中“一切皆对象”是什么意思？
- 变量只是**对象的引用**
- 对象由三部分组成：
  - `id`（身份）
  - `type`（类型）
  - `value`（值）

```python
a = 1
b = a
````

* `a` 和 `b` 指向同一个对象

---

### 2. Python 中变量赋值的本质

* **不是拷贝值**
* 而是 **绑定对象引用**

```python
a = [1, 2]
b = a
b.append(3)
```

---

### 3. `is` vs `==`

* `is`：是否为**同一对象**
* `==`：值是否相等

```python
a = [1,2]
b = [1,2]
a == b   # True
a is b   # False
```

---

### 4. 小整数缓存机制（常考）

```python
a = 100
b = 100
a is b   # True
```

* CPython 对 `[-5, 256]` 做了对象缓存
* **实现细节，不是语言规范**

---

## 二、可变性（Mutability）🔥

### 5. 可变对象 vs 不可变对象

| 类型                        | 是否可变 |
| ------------------------- | ---- |
| int / float / str / tuple | ❌    |
| list / dict / set         | ✅    |

---

### 6. 为什么不可变对象“更安全”？

* 不可变对象：

  * 可 hash
  * 可作为 dict key
* 可变对象：

  * 修改会影响所有引用

---

### 7. 函数参数传递的本质

👉 **对象引用传递**

```python
def f(x):
    x.append(1)

a = []
f(a)
```

* 修改的是对象本身
* 不是重新绑定变量

---

### 8. 默认参数陷阱（送命题）

```python
def f(x, arr=[]):
    arr.append(x)
    return arr
```

* 默认参数在 **函数定义时创建**
* 多次调用共享同一个对象

✅ 正确写法：

```python
def f(x, arr=None):
    if arr is None:
        arr = []
```

## 三、内存管理（Memory Management）

### 9. Python 的内存管理三板斧

1. **引用计数**
2. **标记-清除**
3. **分代回收**

---

### 10. 引用计数机制

* 每个对象维护一个引用计数
* 为 0 时立即释放

```python
a = []
b = a
del a
del b  # 此时对象才释放
```

---

### 11. 循环引用问题

```python
a = []
b = []
a.append(b)
b.append(a)
```

* 引用计数 ≠ 0
* 由 **GC 标记-清除** 解决

---

### 12. 分代回收（为什么快）

* 对象分为：

  * 新生代
  * 中生代
  * 老年代
* **大多数对象“朝生夕死”**

---

### 13. 什么时候 GC 不工作？

* 存在 `__del__` 方法的循环引用
* 可能导致内存泄漏（高级点）

---

## 四、并发模型（Concurrency Model）🔥🔥

### 14. GIL 是什么？

* **Global Interpreter Lock**
* 同一时间只允许一个线程执行 Python 字节码
* 目的是：

  * 简化内存管理
  * 保证解释器线程安全

---

### 15. GIL 带来的影响

| 场景     | 影响      |
| ------ | ------- |
| CPU 密集 | ❌ 多线程无用 |
| IO 密集  | ✅ 影响不大  |

---

### 16. 线程 / 进程 / 协程对比

| 模型 | 是否并行 | 切换成本 | 适合场景   |
| -- | ---- | ---- | ------ |
| 线程 | ❌    | 中    | IO     |
| 进程 | ✅    | 高    | CPU    |
| 协程 | ❌    | 极低   | 高并发 IO |

---

### 17. asyncio 协程的本质

* 单线程
* 事件循环
* **协作式调度**
* `await` 主动让出控制权

---

### 18. 为什么 FastAPI + asyncio 快？

* ASGI 架构
* 非阻塞 IO
* 高并发连接数低开销
